# Python >=3.12 Bump and Ruff `UP` Adoption Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Align the declared Python floor with the version CI actually tests (3.12), then adopt ruff's `UP` rule set to modernize type annotations.

**Architecture:** Two independent commits. The first changes four version declarations so `pyproject.toml`, `Dockerfile`, `README.md`, and `uv.lock` all agree with the CI workflow's existing 3.12. The second adds `UP` to ruff's `select` and applies the resulting auto-fixes across `src/` and `tests/`. No runtime behavior changes in either.

**Tech Stack:** Python 3.12, uv, ruff 0.13.0 (lint + format), mypy, pytest, pydantic 2.11.7, Docker.

## Global Constraints

- Target Python floor is exactly `>=3.12`. Not 3.11, not 3.13.
- Docker base image is `python:3.12-slim` — it must match the CI version, not exceed it.
- `.github/workflows/ci.yml` must NOT be modified. It is already on 3.12 and is the source of truth the other files align to.
- Files under `docs/superpowers/` must NOT be modified. They are dated records of past decisions.
- No behavioral change to server, tools, handlers, client, or models. Every diff is a version declaration or a mechanical annotation rewrite.
- Commit with `uv run git commit`, never bare `git commit`. The `check-no-emojis` pre-commit hook fails with `Executable python not found` outside the uv environment.
- Do not add an explicit `target-version` key to `[tool.ruff]`. Ruff infers it from `requires-python`; Task 1 makes it `py312` implicitly.
- Local ruff is 0.13.0 and the `.pre-commit-config.yaml` hook pins `rev: v0.13.0`. Do not upgrade either without upgrading both — the config file's own comment explains why version drift causes format churn.

### Task 2: Adopt ruff `UP` and apply the modernization

**Files:**
- Modify: `pyproject.toml:28`
- Modify: 36 files under `src/gramps_mcp/` (by `ruff --fix`, not by hand)
- Modify: `tests/test_complete_workflow.py`, `tests/test_search_basic.py` (by `ruff --fix`, not by hand)
- Test: no test file logic changes; verification is the existing offline suite

**Interfaces:**
- Consumes: `requires-python = ">=3.12"` from Task 1. Without it, ruff infers `py310` and rule UP017 does not fire.
- Produces: nothing that later tasks depend on. This is the final task.

Do not hand-edit annotations. Every code change in this task is produced by
`ruff --fix` and `ruff format`. If you find yourself typing `str | None`
manually, you have gone off-plan.

- [ ] **Step 1: See what `UP` will change before enabling it**

```bash
uv run ruff check --select UP --statistics src/ tests/
```

Expected, as measured:

```
202  UP045  non-pep604-annotation-optional   fixable
158  UP006  non-pep585-annotation            fixable
 46  UP035  deprecated-import                not fixable
  4  UP017  datetime-timezone-utc            fixable
  2  UP007  non-pep604-annotation-union      fixable
  1  UP015  redundant-open-modes             fixable
```

413 total. The 46 UP035 findings are reported as not fixable, but they resolve
anyway: once UP006 and UP045 rewrite the annotations, the `from typing import
Dict, List, Optional` lines become unused and F401 — already in `select` —
deletes them. Step 5 confirms this rather than trusting it.

UP017 appears only because Task 1 landed. `datetime.UTC` is a 3.11+ alias, so
at `py310` this rule reports zero.

- [ ] **Step 2: Add `UP` to the ruff rule selection**

In `pyproject.toml`, line 28, replace:

```toml
select = ["E", "F", "I"]
```

with:

```toml
select = ["E", "F", "I", "UP"]
```

- [ ] **Step 3: Apply the fixes**

```bash
uv run ruff check --fix src/ tests/
```

Expected: ruff reports the fixed count and leaves no unfixed `UP` findings. No
`--unsafe-fixes` flag is needed — every applied fix is classified safe.

- [ ] **Step 4: Reformat**

```bash
uv run ruff format src/ tests/
```

Expected: roughly 18 files reformatted. This step is mandatory, not cosmetic.
The UP rewrites shorten annotation lines (`Optional[Dict]` becomes `dict |
None`), which leaves wrapped call signatures that the formatter collapses.
Skipping it makes `ruff format --check` fail in CI.

- [ ] **Step 5: Verify zero residual findings**


If any UP035 findings survive here, the F401 cleanup did not happen as
