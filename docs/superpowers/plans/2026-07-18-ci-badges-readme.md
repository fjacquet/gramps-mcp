# CI/CD, Badges, and README Improvements Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a CI workflow (lint, format check, type check, offline tests with coverage), wire up Codecov, add a badge row to the README, and add a table of contents plus a short Development section.

**Architecture:** One new GitHub Actions workflow file mirrors the commands already used in local development and pre-commit; README changes are three independent, targeted edits to a single file; Codecov activation is a manual step the user performs after the workflow first runs (documented, not automated).

**Tech Stack:** GitHub Actions, `astral-sh/setup-uv`, `codecov/codecov-action`, `pytest-cov`, uv, ruff, mypy.

Spec: `docs/superpowers/specs/2026-07-18-ci-badges-readme-design.md`

## Global Constraints

- Run everything through uv: `uv run pytest`, `uv run ruff`, etc. Commit via `uv run git commit` (pre-commit hooks are installed and will run: ruff, ruff-format, copyright-notice on `.py` files, file-length check, no-emoji check on all files).
- No emojis anywhere, including in `.github/workflows/ci.yml` and the README additions (the no-emoji pre-commit hook checks all files, not just `.py`).
- Never `git add -A` or `git add .` — the working tree contains the user's unrelated WIP (`docker-compose.yml` modified, `docker/` untracked). Stage only the files each task names.
- CI runs exactly these four offline-safe test files, nothing else:
  `tests/test_merge.py tests/test_config.py tests/test_client_merge.py tests/test_utils.py`
- CI targets Python 3.12 only (deliberate choice, not a placeholder for a matrix).
- Badge URLs point at `fjacquet/gramps-mcp` (this fork), matching where the workflows actually run.
- The Codecov upload step must not fail the whole CI run if the token is missing: `fail_ci_if_error: false`.
- The Codecov account/secret setup (codecov.io signup, `CODECOV_TOKEN` GitHub secret) is a manual, human-only step — no task in this plan can perform it. It is called out at the end of this plan, not hidden inside a task.

### Task 2: Add the CI workflow

**Files:**
- Create: `.github/workflows/ci.yml`

**Interfaces:**
- Consumes: `pytest-cov` from Task 1 (the `--cov` flag requires it to be installed).
- Produces: a GitHub Actions workflow named `CI` in file `ci.yml` — Task 3's README badge references this exact path (`.github/workflows/ci.yml` → badge URL `.../actions/workflows/ci.yml/badge.svg`).

- [ ] **Step 1: Create the workflow file**

Create `.github/workflows/ci.yml` with exactly:

```yaml
name: CI

on:
  push:
    branches:
      - main
  pull_request:

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Install uv
        uses: astral-sh/setup-uv@v6
        with:
          python-version: "3.12"
          enable-cache: true

      - name: Install dependencies
        run: uv sync --locked --all-extras --dev

      - name: Lint with ruff
        run: uv run ruff check src/

      - name: Check formatting with ruff
        run: uv run ruff format --check src/ tests/

      - name: Type check with mypy
        run: uv run mypy src/gramps_mcp --ignore-missing-imports

      - name: Run offline tests with coverage
        run: uv run pytest --cov=src/gramps_mcp --cov-report=xml tests/test_merge.py tests/test_config.py tests/test_client_merge.py tests/test_utils.py

      - name: Upload coverage to Codecov
        uses: codecov/codecov-action@v5
        with:
          token: ${{ secrets.CODECOV_TOKEN }}
          fail_ci_if_error: false
```

- [ ] **Step 2: Validate the YAML is well-formed**

```bash
uv run --with pyyaml python -c "import yaml; yaml.safe_load(open('.github/workflows/ci.yml')); print('valid yaml')"
```

Expected: `valid yaml`.

- [ ] **Step 3: Re-run every command the workflow will run, locally, to confirm they pass at this exact commit**

```bash
uv sync --locked --all-extras --dev
uv run ruff check src/
uv run ruff format --check src/ tests/
uv run mypy src/gramps_mcp --ignore-missing-imports
uv run pytest --cov=src/gramps_mcp --cov-report=xml tests/test_merge.py tests/test_config.py tests/test_client_merge.py tests/test_utils.py
```

Expected: `uv sync` reports the lock is unchanged (nothing to do); ruff check clean; ruff format check clean; mypy `Success: no issues found`; pytest ends with `18 passed` and writes `coverage.xml` in the repo root.

- [ ] **Step 4: Do not commit the generated coverage artifacts**

`coverage.py` (via `pytest-cov`) writes both a `coverage.xml` report and a
`.coverage` data file to the repo root when `--cov` is used; neither is
currently in `.gitignore`.

```bash

grep -q "^\.coverage$" .gitignore || echo ".coverage" >> .gitignore
rm -f coverage.xml .coverage
```

Expected: `.gitignore` gains both entries (appended only if not already
present); both generated files from Step 3 are removed.

- [ ] **Step 5: Commit**

```bash

## Post-plan manual step (cannot be automated by any task)

After this branch is pushed to `fjacquet/gramps-mcp` and the CI workflow has
run at least once on `main`:

1. Sign in to [codecov.io](https://codecov.io) with the GitHub account that
   owns the fork.
2. Add the `fjacquet/gramps-mcp` repository in Codecov.
3. Copy the repository upload token Codecov shows.
4. In the GitHub repo: **Settings → Secrets and variables → Actions → New
   repository secret**, name it `CODECOV_TOKEN`, paste the token, save.

Until this is done, the Codecov upload step in CI will report a non-fatal
failure (`fail_ci_if_error: false`, so the rest of CI still passes) and the
Codecov badge will read "unknown" instead of a percentage.
